3. TABEL ENTITAS DATABASE
================================================================================
Proyek menggunakan database relasional mini berbasis file (SQLite) untuk mencatat 
riwayat deteksi secara real-time.

Nama Tabel: engagement_logs
Deskripsi : Menyimpan data rekaman hasil analisis emosi dan tingkat keterlibatan 
            user untuk setiap frame yang berhasil dikirim ke server.

+------------------+--------------+-------------+-------------------------------+
| Nama Kolom (Field)| Tipe Data    | Constraint  | Deskripsi                     |
+------------------+--------------+-------------+-------------------------------+
| id               | INTEGER      | PRIMARY KEY | Auto Increment identifier     |
|                  |              | AUTOINCREMENT| untuk baris data.             |
+------------------+--------------+-------------+-------------------------------+
| timestamp        | TEXT         | NOT NULL    | Waktu penyimpanan data dalam  |
|                  |              |             | format ISO 8601 standar       |
|                  |              |             | (YYYY-MM-DDTHH:MM:SS.mmmmmm). |
+------------------+--------------+-------------+-------------------------------+
| session_id       | TEXT         | NOT NULL    | ID Sesi acak unik (di-generate|
|                  |              |             | oleh frontend) untuk melacak  |
|                  |              |             | riwayat per pengguna/sesi.    |
+------------------+--------------+-------------+-------------------------------+
| emotion          | TEXT         | NOT NULL    | Label emosi terdeteksi        |
|                  |              |             | (happy, sad, neutral, dll).   |
+------------------+--------------+-------------+-------------------------------+
| engagement_score | REAL         | NOT NULL    | Nilai desimal tingkat fokus   |
|                  |              |             | user (0.2, 0.8, atau 1.0).    |
+------------------+--------------+-------------+-------------------------------+

4. ENTITY RELATIONSHIP DIAGRAM (ERD)
================================================================================
Secara logis, sistem ini memisahkan entitas "Session" (Sesi Uji Coba) dan 
"Engagement Log" (Catatan Log Deteksi). Hubungan relasinya adalah satu sesi 
bisa memiliki banyak log deteksi (1 to Many).

    +-------------------+                  +------------------------+
    |      SESSION      |                  |     ENGAGEMENT_LOG     |
    +-------------------+                  +------------------------+
    | PK | session_id   | 1              M | PK  | id               |
    +-------------------+------------------+-----+------------------+
    |    | start_time   |                  | FK  | session_id       |
    |    | ip_address   |                  |     | timestamp        |
    +-------------------+                  |     | emotion          |
                                           |     | engagement_score |
                                           +------------------------+

*Keterangan:*
- PK = Primary Key
- FK = Foreign Key
- Relasi: Satu (1) `SESSION` dapat merekam banyak (M) baris data `ENGAGEMENT_LOG`.
