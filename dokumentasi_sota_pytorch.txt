DOKUMENTASI PROYEK: MIGRASI & OPTIMASI EMOTION RECOGNITION (PYTORCH SOTA)

Dokumen ini menjelaskan keseluruhan proses, arsitektur, dataset, dan antarmuka dari sistem pengenalan emosi SAMAYA yang telah dimigrasikan dan dioptimalkan dari TensorFlow ke PyTorch CUDA untuk dijalankan pada GPU lokal.

================================================================================
1. KONTEKS PROYEK & TUJUAN MIGRASI
================================================================================
* Latar Belakang:
  Awalnya, proyek ini menggunakan framework TensorFlow. Namun, mulai versi 2.11, TensorFlow tidak lagi mendukung akselerasi GPU secara native pada sistem operasi Windows. Hal ini menyebabkan pelatihan model lokal berjalan sangat lambat di CPU.
* Tujuan:
  Memindahkan seluruh alur kerja pelatihan dan inferensi ke PyTorch CUDA secara native di Windows untuk memanfaatkan daya komputasi GPU lokal (NVIDIA GeForce RTX 4060 Laptop GPU) secara maksimal, sekaligus mengoptimalkan akurasi model emosi hingga menembus batas State-of-the-Art (SOTA).

================================================================================
2. MODEL YANG DIGUNAKAN (ARCHITECTURE)
================================================================================
* Backbone Model: EfficientNet-B2
  Dipilih sebagai upgrade dari model baseline sebelumnya (MobileNetV2 dan EfficientNet-B0). EfficientNet-B2 memiliki kapasitas parameter yang lebih tinggi (~9 Juta parameter) sehingga lebih sensitif dalam menangkap gerakan mikro otot wajah yang merepresentasikan emosi, namun tetap ringan dan efisien untuk dijalankan secara real-time pada kartu grafis laptop.
* Classifier Head:
  Dipasang di atas backbone konvolusi, terdiri dari:
  - Dropout layer dengan probabilitas 0.4 (untuk mencegah overfitting).
  - Linear Layer (Fully Connected) yang memetakan fitur ke 7 kelas emosi target.
* Kelas Emosi Target (7 Kelas):
  ['surprise' (terkejut), 'fear' (takut), 'disgust' (jijik), 'happy' (senang), 'sad' (sedih), 'angry' (marah), 'neutral' (netral)]
* Optimasi Presisi:
  Mengaktifkan Automatic Mixed Precision (AMP) menggunakan `torch.amp.autocast` dan `torch.amp.GradScaler` untuk melatih model pada presisi FP16 secara dinamis. Ini menurunkan penggunaan VRAM hingga ~40% dan mempercepat proses pelatihan hingga 2x lipat.

================================================================================
3. DATASET YANG DIGUNAKAN & MANAJEMEN MEMORI
================================================================================
Proyek ini mengombinasikan dua dataset wajah utama:
* FER-2013 (Tahap Pra-pelatihan):
  Dataset ekspresi wajah skala besar berisi ~35.000 gambar. Digunakan pada Tahap 1 untuk melatih model memahami fitur visual wajah umum sebelum melakukan spesialisasi emosi.
* RAF-DB (Tahap Fine-Tuning):
  Dataset ekspresi wajah dunia nyata berisi ~15.339 gambar (~12.271 data latih & ~3.068 data uji). Digunakan sebagai target akhir optimasi.
* Manajemen Memori & Optimasi Loader:
  - In-Memory Raw Bytes Caching: Membaca file gambar mentah berbentuk JPEG ke dalam RAM sebagai bytes terkompresi (bukan float32 tensor). Ini memangkas konsumsi RAM dari ~20 GB menjadi hanya ~100-300 MB dan menghindari masalah disk-space/lockfile di Windows/WSL.
  - Multi-processing DataLoader: Menggunakan `num_workers = 4` untuk mendecode dan mengaugmentasi data di CPU secara paralel di latar belakang selagi GPU melakukan kalkulasi matriks, menghilangkan bottleneck CPU.
  - Random Oversampling: Menyeimbangkan distribusi sampel emosi latih secara dinamis sebelum di-cache.

================================================================================
4. MEKANISME PELATIHAN (TRAINING PIPELINE)
================================================================================
Proses pelatihan dibagi menjadi dua tahap utama yang berjalan secara berurutan:

* TAHAP 1: PRE-TRAINING FER-2013 (15 Epochs)
  Melatih model EfficientNet-B2 dari ImageNet pada dataset FER-2013 untuk mentransfer pengetahuan fitur wajah emosi. Bobot terbaik disimpan di file `fer2013_efficientnetb2_pytorch.pth`.

* TAHAP 2: FINE-TUNING RAF-DB (35 Epochs)
  Memuat bobot dari Tahap 1 dan melatihnya dalam 3 Fase bertahap pada dataset RAF-DB:
  - Fase 2.1: Training Custom Head Only (10 Epochs, LR = 1e-3, Backbone dibekukan).
  - Fase 2.2: Fine-Tuning Top Layers (15 Epochs, unfreeze layer atas features index >= 7). Menggunakan Discriminative Learning Rates (Backbone LR = 5e-5, Head LR = 1e-4).
  - Fase 2.3: Fine-Tuning All Layers (10 Epochs, unfreeze seluruh model). Menggunakan Discriminative Learning Rates (Backbone LR = 1e-5, Head LR = 1e-5).

* Kunci Keberhasilan & Stabilitas Pelatihan:
  - Optimizer AdamW: Menggunakan decoupled weight decay (`1e-2`) untuk mencegah overfitting dan meningkatkan kemampuan generalisasi pada data uji.
  - Gradient Clipping (Norm = 1.0): Mencegah terjadinya gradien meledak (*gradient explosion*) saat unfreezing backbone konvolusi.
  - Proteksi Regresi Checkpoint: Mengevaluasi akurasi validasi model sebelum fase baru dimulai (`initial_acc`) dan menjadikannya baseline akurasi minimum. Hal ini menjamin file model tidak akan pernah tertimpa oleh bobot dengan performa lebih rendah di fase berikutnya.
  - Test-Time Augmentation (TTA): Di akhir evaluasi, prediksi dihitung dari rata-rata citra asli dan citra yang dibalik secara horizontal (*horizontal flip*).

* Hasil Performa Akhir:
  - Akurasi Validasi Terbaik: 79.76%
  - Akurasi Akhir dengan TTA: 80.15% (Meningkat +11.73% dari baseline awal MobileNetV2 68.42%).
  - File Bobot SOTA Tersimpan: `samaya_rafdb_sota_pytorch_b2_adamw.pth`

================================================================================
5. TAMPILAN ANTARMUKA & INTEGRASI APLIKASI (WEB USER INTERFACE)
================================================================================
Model yang telah dilatih kemudian dideploy ke dalam aplikasi web terintegrasi:

* Backend API (FastAPI - `main.py`):
  - Secara dinamis mendeteksi keberadaan file model PyTorch SOTA (`samaya_rafdb_sota_pytorch_b2_adamw.pth`).
  - Jika terdeteksi, backend memuat model menggunakan PyTorch secara langsung (dan melewati pemuatan TensorFlow/Keras untuk menghemat memori GPU/RAM).
  - Menyediakan endpoint POST `/predict` untuk menerima frame gambar wajah dari frontend, memprosesnya melalui preprocessing PyTorch, melakukan inferensi emosi secara cepat, dan menghitung skor keterlibatan (*Engagement Score*).
  - Mengintegrasikan penyimpanan log emosi ke database SQLite (`database.py`).
  - Mendukung callback untuk autentikasi Zoom OAuth/App.

* Frontend Dashboard (`index.html`):
  - Interface Dashboard yang modern dan premium dengan skema gelap (dark-mode), visualisasi Chart.js, dan status logger.
  - Mengaktifkan "Live Camera Input" di sisi browser klien.
  - Menggunakan library MediaPipe Face Detection di sisi frontend untuk melacak koordinat wajah secara real-time langsung di browser (Edge AI).
  - Memotong (*crop*) wajah secara otomatis di browser dan mengirimkan potongan citra wajah tersebut ke backend `/predict` untuk meminimalkan beban bandwidth jaringan.
  - Menampilkan grafik keterlibatan (*Engagement Score Chart*) secara dinamis:
    - Emosi *happy* & *surprise* = 1.0 (High Engagement)
    - Emosi *neutral* = 0.8 (Focused)
    - Emosi lainnya (sad, angry, fear, disgust) = 0.2 (Cognitive Overload/Frustrated)
  - Menampilkan tabel log aktivitas emosi real-time di bagian bawah dashboard.
